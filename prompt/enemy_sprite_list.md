# 像素怪物角色 - 动画序列帧提示词

## 概述

本文档包含 5 种怪物的动画序列帧生成提示词，每种怪物包含 3 个动作：
- **行走 (Walk)**: 5 帧循环
- **攻击 (Attack)**: 5 帧循环
- **死亡 (Death)**: 5 帧一次性播放

---

## 生成参数（通用）

| 参数 | 值 |
|------|-----|
| 生成方式 | image2image（以参考图为布局基准） |
| 参考图 | `seedance-input/rule.png` |
| ratio | 16:9 |
| resolution_type | 2k |
| model_version | 5.0 |
| poll | 120 |

> 输出原始尺寸约 2560×1440（16:9），后续裁切为 5 张独立帧并缩放到目标尺寸。

### 参考图说明 (`seedance-input/rule.png`)

布局模板图（2560×1440），白底上有 5 个等距黑色矩形区域，定义了角色的精确放置位置：

| 属性 | 值 |
|------|-----|
| 黑色区域尺寸 | 400×650 像素（每个） |
| 顶部留白 | 427 像素 |
| 底部留白 | 363 像素 |
| 左侧留白 / 矩形间距 | 112 像素 |
| 黑色区域行范围 | y=427 ~ y=1076 |

生成时角色必须**完全填满**每个黑色矩形区域，黑色区域外保持纯白背景。

---

## 核心约束（所有动作必须遵守）

```
⚠️ 重要：以下约束必须在生成时严格执行（基于参考图 seedance-input/rule.png，输出尺寸 2560×1440）

1. 【布局参考】以 seedance-input/rule.png 为布局基准，图中有 5 个 400×650 像素的黑色矩形区域。
   生成结果中，这 5 个矩形区域必须完全被角色像素填满，区域外为纯白背景。
   ★ 不要在提示词中提及"覆盖黑色区域"，改为描述"角色填满整个帧区域"★

2. 【统一角色比例】所有 5 帧中角色必须保持完全相同的像素大小，
   角色从头顶到脚底填满帧区域的高度，宽度填满帧区域的宽度。
   三个动作（walk/attack/death）之间的角色大小也必须完全一致。

3. 【脚底锚点】所有 5 帧角色的脚底必须处于同一水平高度，
   距图片底边 363 像素处，不得上下偏移。

4. 【头顶锚点】角色头顶对齐帧区域的顶边，距图片顶边 427 像素处，所有帧保持一致。

5. 【帧布局】5 帧从左到右排列，帧与帧之间无任何分隔线、边框、标注或辅助线，
   背景保持纯净纯白，禁止出现横线、竖线、分割线、网格线、标尺、刻度。

6. 【角色位置固定】★ 核心约束 ★ 所有 5 帧中角色必须严格固定在各自帧区域的
   正中央位置（X轴Y轴均固定），仅允许动作姿态变化，
   绝对禁止角色位置在帧与帧之间出现任何偏移、漂移或位置不一致的问题。

7. 【禁止辅助元素】★ 画面中不得出现任何非角色内容的辅助性视觉元素 ★
   包括：参考线、对齐线、网格、标注文字、箭头、尺寸标记、黑边、黑框。
   整张图片只包含纯白背景和 5 个角色帧，不含任何额外的图形元素。

8. 【边缘处理】★ 防止黑边的关键 ★
   - 角色轮廓边缘不得有深色描边、阴影或轮廓线
   - 角色最外层像素必须直接衔接纯白背景，不能有深色边缘残留
   - 如果角色有深色服装/铠甲，最外层像素之后立即是白色，不能有深色过渡
```

## CLI 模板

```bash
dreamina image2image \
  --images seedance-input/rule.png \
  --prompt="[提示词]" \
  --ratio=16:9 \
  --resolution_type=2k \
  --model_version=5.0 \
  --poll=120
```

## 一、Slime（史莱姆）

**视觉设定**: 体型娇小的圆滚滚凝胶状生物，亮绿色半透明果冻质感身体，从正上方俯视可见扁圆形躯体顶面，两颗圆鼓鼓的白色小眼睛镶嵌在顶部，无明显四肢，表面有轻微光泽反光。体型对应游戏中最小单位（碰撞半径 15px）。

## 二、Skeleton（骷髅弓手）

**视觉设定**: 中型骷髅远程怪，从正上方俯视可见泛黄的圆形头骨顶部，肋骨隐约可见，持有一把骨制小弓，关节之间有暗色腐朽痕迹，整体泛黄白骨色调，散发着阴冷气息。体型居中（碰撞半径 18px）。

---

### 行走 (Walk)

## 后处理：裁切脚本

生成的 sprite sheet（约 2560×1440）需裁切为 5 张独立帧，并缩放到目标尺寸。透明背景 PNG。
> 输出目录 client/Assets/Sprites/Enemy/<monster_name>

```bash
python3 scripts/split_sprite_sheet.py <sprite_sheet.png> <monster_name>_<action>
```

裁切逻辑（与弓箭手相同）：
1. 将 sprite sheet 等分为 5 个区域（每区域约 512×1440）
2. 每帧去除白色背景（阈值 > 240 视为白色，转为透明）
3. 裁剪到角色内容边界框 + 5px 内边距
4. **验证脚底对齐**：若各帧裁剪后内容高度差异 > 5px，则以脚底为锚点统一对齐
5. 保持宽高比缩放至目标尺寸，使用最近邻插值（NEAREST）保留像素锐度
6. 居中放置在透明画布上
